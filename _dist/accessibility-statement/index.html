Accessibility Statement

According to the World Wide Web Consortium (WC3)’s Web Accessibility Initiative (WAI), web accessibility is an essential part of building any website. WAI has three different areas for which it develops web accessibility standards, at three different levels--development ( Authoring Tool Accessibility Guidelines), content management (Web Content Accessibility Guidelines), and user experience (User Agent Accessibility Guidelines) For my purposes, I will be focusing on the standards of the Web Content Accessibility Guidelines (WCAG). Principles and their underlying guidelines are presented and summarized below; after each guideline is summarized, I will illustrate how this website meets it.

WCAG rests on four guiding principles: accessible websites must be “perceivable,” “operable,” “understandable,” and “robust.” The principle of perceivability relies on text alternatives for images and video, adaptability (“create content that can be presented in different ways [for example simpler layout] without losing information or structure”), and distinguishability (“make it easier for users to see and hear content including separating foreground from background”). This website enacts the principle of perceivability by being adaptable, in that the image on the homepage is captioned with an image description, and the information on the drop-down menu is reproduced in full and without any major formatting in a table of contents on the “MLIS portfolio” page; the content is made distinguishable by virtue of the simplicity of the menu and page contents and the high color contrast, with black on a white background.

The principle of operability relies on keyboard accessibility (all aspects of the website are accessible via keyboard), enough time (“provide users enough time to read and use content”), navigability (users are able to orient themselves within the site, and can find content easily), and input modalities (“Make it easier for users to operate functionality through various inputs beyond keyboard”). This website takes these guidelines into account by having a separate page (“Table of contents") that is keyboard accessible (unlike the dropdown menu, which relies on hovering with a mouse); forgoing the enough time guideline by having no time-sensitive aspects while maintaining interactivity; maintaining navigability by giving each page a clear title (along with subheadings on some pages), as well as giving each link a clearly defined purpose within the link text itself (all link text indicates the purpose of the link, without needing additional context. Because assistive technology often provides users with a list of links, the link text is itself very important); and keeps input modalities simple by having no multipoint gestures exist on the website.

The principle of understandability relies on readability (“make text content readable and understandable”), predictability (pages appear and can be used in ways that are easily anticipated), and input assistance (“help users avoid and correct mistakes”). This website is readable in that it tries to keep use of jargon and specialized language to a minimum, where possible; predictable in that it is structured like many other websites with a homepage and drop-down menu; and forgoes input assistance by not requiring input from users.

The final principle, robustness, means that “content must be robust enough that it can be interpreted by a wide variety of user agents, including assistive technologies.” There is only one guideline for this principle, compatibility, which entails ensuring that current and future user agents will be able to use the site, especially with respect to assistive technologies. This website will grow and change according to my own career trajectory, and I will continue to update and change it, with with WCAG principles at the forefront of my mind.

Making a website always entails making an accessible website. The importance of thinking through how content can be perused and taken in in different ways (using assistive technologies, without a mouse, etc.) ensures that the design of this website is not prohibitive for any users whatsoever. If there is something that I have overlooked or omitted with regards to accessibility, please contact me.
